# 前列腺MRI 3D分割系统使用指南

## 概述

本系统是一个基于 Next.js 的交互式医学图像分割 Web 应用，支持对前列腺 MRI 体积进行 3D 分割。系统使用 Medical SAM2 模型进行自动分割，支持两种标注类型：
- **WG** (Whole Gland): 整体前列腺标注
- **CG** (Central Gland): 中央腺体标注
- **PZ** (Peripheral Zone): 外周带（由系统自动生成）

## 文件格式要求

### 支持的文件格式

**仅支持 `.nii.gz` 格式的 NIfTI 文件**

这是 MRI 医学影像的标准格式，必须是 gzip 压缩的 NIfTI 文件。

**正确的文件名示例：**
- `prostate_mri.nii.gz`
- `patient_001_axial.nii.gz`
- `mri_volume_t2.nii.gz`

**不支持的文件格式：**
- ❌ `.nii` (未压缩的 NIfTI)
- ❌ `.dcm` (DICOM 格式)
- ❌ `.jpg`, `.png`, `.tiff` (图像文件)
- ❌ `.zip`, `.rar` (压缩文件)

### 如何获取 `.nii.gz` 文件

1. **从医学影像处理软件导出**：
   - 使用 ITK-SNAP、3D Slicer 等软件
   - 选择 "Export to NIfTI (.nii.gz)"

2. **从 DICOM 转换**：
   - 使用 `dcm2niix` 工具将 DICOM 序列转换为 NIfTI
   - 命令：`dcm2niix -o output_dir input_dicom_folder`

3. **下载公开数据集**：
   - PROSTATEx Challenge
   - NCI-ISBI 2013 Challenge
   - PROMISE12 Challenge

## 使用步骤

### 1. 上传 MRI 体积

1. 点击 "上传 NIfTI 文件" 按钮
2. 选择一个 `.nii.gz` 文件
3. 等待系统加载体积（可能需要几秒到几分钟，取决于文件大小）

**注意事项：**
- 文件大小建议不超过 500MB
- 如果文件较大（>500MB），系统会提示但仍然允许上传
- 系统会自动检测文件格式，如果格式不正确会提示错误

### 2. 浏览切片

上传成功后，你可以：

- **手动导航**：使用左右箭头按钮（< 和 >）切换切片
- **滑块导航**：拖动滑块快速跳转到任意切片
- **自动播放**：点击播放按钮（▶）自动播放所有切片
  - 可以通过滑块调整播放速度
  - 点击暂停按钮（⏸）停止播放

**切片信息显示：**
- 体积大小：D x H x W（深度 x 高度 x 宽度）
- 当前切片：显示当前是第几片，共多少片

### 3. 绘制标注框

系统支持两种标注类型：

#### 标注类型选择
- **WG (Whole Gland)**: 整体前列腺（绿色标注框）
- **CG (Central Gland)**: 中央腺体（蓝色标注框）

#### 绘制步骤
1. 选择标注类型（WG 或 CG）
2. 在切片上拖动鼠标绘制矩形框
3. 包围前列腺或感兴趣区域
4. 松开鼠标完成标注

#### 标注管理
- **删除单个标注**：点击标注框右上角的删除按钮（🗑️）
- **清空所有标注**：点击 "清空标注" 按钮
- **重置系统**：点击 "重置" 按钮清空所有内容

**提示：**
- 可以在同一切片上绘制多个标注
- 可以在不同切片上绘制标注
- 建议在前列腺最清晰的切片上绘制 WG 标注
- CG 标注应该位于前列腺中央

### 4. 执行分割

1. 确保已上传体积并绘制至少一个标注框
2. 点击 "运行分割" 按钮
3. 等待系统处理（可能需要 1-5 分钟）
4. 分割完成后，结果会显示在当前切片上

**分割结果：**
- **WG (Whole Gland)**: 整体前列腺分割（绿色区域）
- **CG (Central Gland)**: 中央腺体分割（蓝色区域）
- **PZ (Peripheral Zone)**: 外周带（黄色区域，自动计算）

## 常见问题

### Q1: 上传文件时提示"文件格式错误"

**原因**：上传的文件不是有效的 `.nii.gz` 文件

**解决方案**：
1. 确认文件扩展名是 `.nii.gz`（不是 `.nii` 或其他）
2. 确认文件是 gzip 压缩的 NIfTI 文件
3. 尝试重新下载或重新导出文件
4. 使用 nibabel 验证文件：
   ```python
   import nibabel as nib
   nii = nib.load('your_file.nii.gz')
   print(nii.shape)
   ```

### Q2: 切片显示方向不正确

**原因**：NIfTI 文件维度顺序不符合预期

**解决方案**：
- 系统会自动检测并转置维度
- 如果仍有问题，检查原始文件的 header 信息

### Q3: 分割结果不准确

**可能原因**：
1. 标注框位置不准确
2. 标注框太大或太小
3. MRI 图像质量不佳

**优化建议**：
1. 重新绘制更精确的标注框
2. 尝试在不同切片上绘制标注
3. 确保标注框完全包围前列腺
4. 对于复杂病例，可能需要多个标注框

### Q4: 加载或分割速度很慢

**可能原因**：
1. 文件太大（>500MB）
2. 系统负载高
3. 网络延迟

**解决方案**：
1. 考虑裁剪体积到感兴趣区域
2. 降低图像分辨率（使用预处理的文件）
3. 在非高峰时段使用系统

### Q5: 页面卡死或无响应

**解决方案**：
1. 刷新浏览器页面
2. 清除浏览器缓存
3. 检查浏览器控制台是否有错误信息

## 技术细节

### 系统架构

- **前端**: Next.js 16 + React 19 + TypeScript
- **后端**: Next.js API Routes + Python (via spawn)
- **AI 模型**: Medical SAM2
- **图像处理**: nibabel, scipy, MONAI

### 数据流

1. 用户上传 `.nii.gz` 文件
2. 前端转换为 base64 编码
3. 后端 Python 脚本解码并加载
4. 提取所有切片并转换为 PNG base64
5. 用户绘制标注框
6. 调用分割 API，传递体积和标注
7. Python 脚本运行 Medical SAM2 推理
8. 返回分割结果（PNG base64）
9. 前端渲染分割结果

### API 端点

- `POST /api/load-volume`: 加载 NIfTI 体积
- `POST /api/segment-3d`: 执行 3D 分割
